# Crypto Reality Visual Rules

## Brand Safety

- Do not use real exchange logos.
- Do not use real crypto project logos.
- Do not use financial advice wording.
- Use fictional tickers only.
- Keep copy game-first: choices, risk, events, survival, archetypes, score, and progression.

## UI Direction

- Use dark-first UI as the default visual mode.
- Keep primary surfaces in the midnight/obsidian range and reserve neon cyan, turquoise, lime, amber, and danger red for state and action accents.
- Prefer sharp sci-fi panels, thin neon borders, compact labels, and high-contrast state feedback.
- Keep cards at `8px` radius or less unless a component has a specific gameplay reason to differ.

## Character Safe Area

- Keep character faces, hands, and key props clear of button rows, captions, and stat overlays.
- Preserve a minimum 10% inset from the image edge for important character details.
- Avoid cropping heads, eyes, weapons/tools, or readable character silhouettes in promo and gameplay cards.
- Do not place high-glow UI over the face area; use darker lower-third or side-safe zones when text must overlay art.

## Typography

- Use one display stack for headings and labels: `Orbitron`, with `Exo 2`, `Avenir Next Condensed`, and system fallbacks.
- Use one UI stack for body and controls: `Inter`, with system fallbacks.
- Use tabular numbers for timers, balances, scores, stats, and leaderboard values.
